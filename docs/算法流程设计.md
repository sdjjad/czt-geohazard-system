# 算法流程

## 1. CF 值计算

确定性系数法(Certainty Factor)，评估各分类区地质灾害易发性。

CF = (PPa - PPs) / (PPa * (1 - PPs))  当 PPa >= PPs
CF = (PPa - PPs) / (PPs * (1 - PPa))  当 PPa < PPs

PPa = 分类区内灾害面积 / 分类区总面积
PPs = 研究区总灾害面积 / 研究区总面积

CF值范围 [-1, 1]，正值=易发，负值=不易发。

### 计算步骤

1. 读取分类面图层全部要素 (ShapefileFeatureTable.OpenAsync)
2. 读取灾害点图层全部要素
3. 如果指定了分级分区方法，先对数值字段重分类
4. 计算研究区总面积: sum(GeometryEngine.AreaGeodetic)，单位km2
5. 计算PPs: hazardCount * 0.05 / totalArea
6. 逐个面要素，用 GeometryEngine.Intersects 判断包含哪些灾害点
7. 按分类名汇总面积和灾害点数
8. 对每个分类计算PPa、CF值
9. 按CF降序排列输出

### 性能

每20个面要素更新一次进度。空间叠加复杂度 O(n*m)，ArcGIS Runtime 内部有空间索引优化。

## 2. 分级分区

对连续数值字段自动分组，用于后续CF分析。四种方法：

### 等间距

```
min = values[0], max = values[n-1], range = max - min
断点[i] = min + range * i / classCount  (i = 0..classCount)
```

### 分位数

```
排序 values
断点[0] = values[0]
断点[i] = values[n * i / classCount]  (i = 1..classCount-1)
断点[classCount] = values[n-1]
去重排序
```

### 标准差

```
mean = avg(values)
stdDev = sqrt(avg((v-mean)^2))
以mean为中心，每0.5σ取断点: mean + k*0.5σ (k = -2.5..2.5)
截断到数据范围内
不足3个断点时回退等间距5级
```

### Jenks 自然间断点

```
目标: 组内方差最小，组间方差最大

1. 排序values
2. 计算SSD矩阵: SSD[i][j] = 区间[i,j]内值到均值的方差和
3. DP: dp[i][k] = 前i个元素分k组的最小总方差
   递推: dp[i][k] = min(dp[j][k-1] + SSD[j+1][i])
4. 回溯提取断点位置
5. 添加首尾值

复杂度: O(n^2 * classCount)
```

### 分类标签生成

```
遍历断点区间, value ∈ [breaks[i], breaks[i+1]) → "12.50 - 25.00"
```

## 3. 按字段配色

1. 读取图层指定字段所有唯一值
2. 判断字段类型: Text → 唯一值渲染; 数值型且>10个唯一值 → 分级渲染
3. 为每个分类分配色带颜色(循环使用)
4. 创建 UniqueValueRenderer 或 ClassBreaksRenderer
5. 应用到图层 featureLayer.Renderer = ...

## 4. 专题制图

### 地图底图

```
EsriMapView.ExportImageAsync() → RuntimeImage
GetEncodedBufferAsync() → PNG字节流 → 保存临时文件 → BitmapImage → 铺满画布
```

### 真实图例

```
获取图层 FeatureLayer.Renderer
  UniqueValueRenderer → 遍历 UniqueValues {Label, Symbol.Color}
  ClassBreaksRenderer → 遍历 ClassBreaks {Label, Symbol.Color}
  SimpleRenderer → 单条 {layer.Name, Symbol.Color}
生成: [16x12色块] + [标签文字] for each item
```

### 比例尺

```
metersPerPixel = mapScale / 96 * 0.0254
barWidth = 100px, barMeters = barWidth * metersPerPixel
>10km: "X km", >1km: "X.X km", <1km: "X m"
绘制: 4格黑白相间 + 距离标注
```

### 指北针

上半黑三角 + 下半白底黑边三角 + N字母。黑白样式。

### PNG导出

```
RenderTargetBitmap(canvasW, canvasH, 96dpi) → Render(MainCanvas) → PngBitmapEncoder → File
```

## 5. 属性编辑保存

1. ShapefileFeatureTable.OpenAsync 打开SHP
2. QueryFeaturesAsync 读取全部要素
3. 构建DataTable(可编辑), 绑定DataGrid
4. 用户双击编辑 → RowState=Modified
5. 保存: 遍历DataRow, 对比原始值, UpdateFeatureAsync写回
6. DataTable.AcceptChanges() 标记已保存

## 6. 属性表生成

1. 清理孤儿CSV(对应SHP已删除的)
2. 递归扫描 spatialRoot/*.shp
3. 每个shp: OpenAsync → 读字段和属性 → 写CSV到attrRoot镜像路径
4. 清理空目录

## 7. 坐标和比例尺

```
比例尺: ViewpointChanged → MapScale → 格式化 → StatusScale.Text
  格式: >1M→"1:X.XM", >1K→"1:XK", else→"1:X"

坐标: MouseMove → 节流100ms → ScreenToLocation → StatusCoord.Text
  格式: "113.0245E  27.8932N"
```
